# Edge Deployment Plan

This document defines the production path for running the syllabus study-plan model on edge devices.

## 1) Device Matrix And Targets

Primary target profile is mid-range mobile (balanced SLA).

| Tier | Representative hardware | Runtime target | Context budget | p95 latency target | Peak RAM target |
|---|---|---|---:|---:|---:|
| Fast SKU | 6-core mobile CPU, 4-6 GB RAM | INT4/INT8 quantized | 1200 tokens | <= 2.5s / chunk | <= 900 MB |
| Balanced SKU | 8-core mobile CPU/NPU, 6-8 GB RAM | INT4/INT8 quantized | 1500 tokens | <= 3.5s / chunk | <= 1.5 GB |
| Fallback CPU-only | low-end ARM CPU | INT4 | 900 tokens | <= 5.0s / chunk | <= 800 MB |

Notes:
- A chunk is a section-sized document slice produced by edge chunking utilities.
- Targets are release gates and should be enforced in CI-ready benchmark reports.

## 2) Model SKUs

- `fast`:
  - Smaller base model tier.
  - Prioritizes startup, memory, and responsiveness.
- `balanced`:
  - Higher quality tier with slightly higher latency footprint.
  - Default for devices that pass startup capability checks.

Runtime chooses SKU from local capability probe:
- Available memory
- CPU score
- Optional NPU/NNAPI availability

## 3) Prompt And Token Policy

- Keep system prompt static and cache tokenized form.
- Section-aware chunking before generation.
- Hard limits:
  - Max 1500 input tokens on balanced SKU
  - Max 1200 input tokens on fast SKU
  - Max 768 output tokens for plan JSON per chunk

## 4) Quality Gates

Required before edge release:
- JSON parse success >= 98% on held-out edge eval set.
- `mean_study_plan_macro_f1` >= +10% over regex-only baseline.
- Schema-critical keys missing rate = 0% (`study_plan`, `document_id`, plan arrays).
- Held-out split has no train/valid domain overlap leakage beyond approved threshold.

## 5) Runtime Safety And Fallback

- Attempt strict JSON decode once.
- On parse failure:
  - Retry one time with stricter instruction suffix.
  - If still invalid, emit deterministic fallback using first-pass entity groups.
- Always emit schema-complete object with empty arrays instead of nulls.

## 6) Telemetry (Privacy-Safe)

Only aggregate counters and buckets:
- Parse success/failure counts
- Latency buckets (`<1s`, `1-2s`, `2-3.5s`, `>3.5s`)
- Output token bucket
- Chunk count bucket

Never upload raw syllabus text, PII fields, or generated content.

## 7) Rollout Checklist

- Benchmark report generated by `edge/benchmark_edge_inference.py`.
- Data quality gates pass via `validate_training_readiness.py`.
- Quantized artifacts exported with manifest (`edge/export_quantized_model.py`).
- Release note includes model hash, quantization method, and gate results.
